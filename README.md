# 開発の方針
基本的にはissue駆動開発とし、
1. タスクに対してissueを立てる
2. 自分が担当する場合issueのassignerに自分を選択
2. それに対応させた名前のブランチを切る
3. PRを送る

のような流れで進めていきたいと思います
